## Project Overview

**ServerDash** — a self-hosted Linux server management dashboard. Each server runs its own independent instance. FastAPI serves both the REST API and the Vue 3 SPA (built as static files into `backend/static/`). No separate web server required.

## Commands

### Backend

```bash
# Install dependencies
pip install -r backend/requirements.txt

# Initialize the database (creates admin user)
python -m backend.scripts.init_db

# Generate self-signed TLS certificate
python -m backend.scripts.generate_cert

# Run development server (HTTPS on port 8443)
python -m backend.main

# Run backend tests
pytest

# Run a single test file
pytest tests/test_auth.py

# Run a single test
pytest tests/test_auth.py::test_login_success
```

### Frontend

```bash
cd frontend

# Install dependencies
npm install

# Dev server (proxies API to localhost:8443)
npm run dev

# Build for production (output goes to ../backend/static/)
npm run build

# Run tests (Vitest)
npm test

# Watch mode
npm run test:watch
```

### Production

```bash
# Start with gunicorn (4 workers, uvicorn worker class, HTTPS)
gunicorn -w 4 -k uvicorn.workers.UvicornWorker backend.main:app \
  --bind 0.0.0.0:8443 --certfile certs/cert.pem --keyfile certs/key.pem

# Or via Procfile
# web: gunicorn -w 4 -k uvicorn.workers.UvicornWorker backend.main:app --bind 0.0.0.0:8443 --certfile certs/cert.pem --keyfile certs/key.pem
```

## Architecture

### Backend (`backend/`)

```
backend/
  main.py              — FastAPI app, mounts routers, serves SPA fallback
  config.py            — Settings via pydantic-settings, reads from .env
  database.py          — SQLAlchemy engine + SessionLocal (SQLite)
  dependencies.py      — get_current_user JWT dependency
  limiter.py           — slowapi rate limiter instance
  models/              — SQLAlchemy ORM models (User, Script, ExecutionLog)
  schemas/             — Pydantic request/response schemas
  routers/             — FastAPI routers (auth, system, services, files, scripts, crontab, logs)
  services/            — Business logic layer (called by routers)
  core/
    logging.py         — Structured logging setup
    health.py          — /health endpoint
  middleware/
    security.py        — Security headers middleware
  scripts/
    init_db.py         — Database initialization + admin user seeding
    generate_cert.py   — Self-signed cert generation
  static/              — Built Vue SPA (git-ignored, generated by npm run build)
```

**API routes** are all prefixed `/api/`:
- `POST /api/auth/login` — returns JWT access token
- `GET /api/system/metrics` — CPU, RAM, disk, uptime, load average
- `GET/POST /api/services` — systemd service management
- `GET/POST /api/files` — filesystem browse/upload/download
- `GET/POST /api/scripts` — script favorites + execution
- `GET/POST /api/crontab` — crontab read/write
- `GET /api/logs/executions` — execution history

**Auth flow**: JWT-only (no refresh token persistence). `get_current_user` dependency reads `Authorization: Bearer <token>`. Token is created on login and refreshed via `POST /api/auth/refresh`.

**Database**: SQLite at `./data/serverdash.db`. Tables auto-created on startup via `Base.metadata.create_all()`. No migrations framework — schema changes require manual handling.

### Frontend (`frontend/src/`)

```
main.js              — App bootstrap: PrimeVue (Aura theme), Pinia, Vue Router
App.vue              — Root layout (sidebar + header + router-view)
router/index.js      — Routes: /, /services, /files, /scripts, /crontab, /logs
stores/
  auth.js            — Pinia store: token, user, login/logout
  scripts.js         — Pinia store: script favorites
api/client.js        — Axios instance with JWT interceptor + base URL
composables/
  usePolling.js      — Simple setInterval-based poller: { start, stop }
views/               — One .vue per route (Dashboard, Services, Files, Scripts, Crontab, Logs)
components/
  layout/            — AppSidebar, AppHeader
  dashboard/         — MetricCard (circular gauge), UptimeDisplay
  services/          — StatusBadge
  files/             — FileList, DirTree
```

**UI library**: PrimeVue 4 with the Aura preset. Dark mode toggled via `[data-theme="dark"]` attribute on the root element.

**Typography**: `--font-mono: 'Fira Code'` for labels/values, `--font-ui: 'Fira Sans'` for body text. Defined in `assets/style.css`.

**Polling**: `usePolling(fn, intervalMs)` wraps `setInterval`. Dashboard polls `/api/system/metrics` every 5s and `/api/logs/executions` every 30s. Polling must be stopped in `onUnmounted`.

## Configuration

All config via `.env` file (see `.env.example`). Key variables:

| Variable | Default | Purpose |
|----------|---------|---------|
| `JWT_SECRET` | `dev-secret-change-in-production` | JWT signing key |
| `ADMIN_USERNAME` | `admin` | Bootstrap admin user |
| `ADMIN_PASSWORD` | `changeme` | Bootstrap admin password |
| `PORT` | `8443` | Server port |
| `DB_PATH` | `./data/serverdash.db` | SQLite path |
| `CERT_FILE` / `KEY_FILE` | `./certs/cert.pem` / `./certs/key.pem` | TLS certs |
| `ALLOWED_ORIGINS` | `["https://localhost:8443"]` | CORS whitelist |

## Key Constraints

- **Single-user system**: only one admin user exists (seeded by `init_db.py`). No user registration.
- **SQLite only**: no migration tooling. Schema changes are manual. New indexes → update `add_indexes.py`.
- **HTTPS required**: the frontend Axios client is configured for the HTTPS base URL. Dev mode uses Vite's proxy.
- **Rate limiting**: login endpoint is limited to 10 requests/minute via slowapi.
- **Background scheduler**: `backend/scheduler.py` runs via FastAPI lifespan. All periodic tasks go here. See `serverdash-performance` skill for the required pattern.
- **LOG_RETENTION_DAYS**: execution logs older than this (default 30) are deleted nightly at 2 AM. Configurable via `.env`.

## Permissions for New Features

Every new feature added to ServerDash MUST follow this checklist:

1. **Identify or create a resource name** (e.g., `backups`, `alerts`)
2. **Define which actions apply** from: `read`, `write`, `delete`, `execute`
3. **Add default permissions** to `DEFAULT_PERMISSIONS` in `backend/scripts/init_db.py` for `operator` and `readonly` roles (admin always has everything — do NOT add admin entries)
4. **Use `require_permission(resource, action)`** in every new backend endpoint — never use bare `get_current_user` for protected resources
5. **Use `auth.hasPermission(resource, action)`** in the frontend to conditionally render action buttons and filter nav items
6. **Add the resource to `MATRIX_ROWS`** in `frontend/src/views/AdminPermissionsView.vue` so the admin can edit it from the UI
7. **Update the default matrix** in `docs/superpowers/specs/2026-04-16-roles-permissions-design.md`

**Never use `get_current_user` directly** in a router that controls access to a resource — always use `require_permission`.

## Development Notes

### Documentation & Logging
- **Always document significant changes**: Each feature, bugfix, or major change must be logged in `/docs/bitacora.md` under the current date. Include: description, files modified, bugs fixed, and important notes.

### Frontend & Chart.js
- **WebSocket proxy in Vite**: Add `ws: true` to vite.config.js proxy config to forward WebSocket connections correctly in dev mode.
- **Chart.js in Vue**: Update charts with `chart.update('none')` instead of reactive properties to avoid unnecessary re-renders. Use `watchEffect` for theme changes.
- **PrimeVue 4 breaking changes**: Card component uses `#content` slot (not `#header` or default). DataTable styling requires deep CSS overrides on cells.

### Backend & Performance
- **Async blocking calls**: Sync calls (psutil, subprocess) in FastAPI handlers must be wrapped with `await run_in_threadpool(...)` to prevent event loop blocking.
- **Network data without root**: Parse `/proc/net/{tcp,tcp6,udp,udp6}` directly for connection data instead of `psutil.net_connections()`, which fails silently without sudo.
